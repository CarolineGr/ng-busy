Ng-Busy

An AngularJS module for reacting to when your app is busy.

About

The core of this module is the busyInterceptor which installs itself in the $http service. It keeps track of when requests begin and end (whether they are rejected or completed successfully) and keeps a running tally of remaining requests. It broadcast events when requests begin and end (busy.begin and busy.end, respectively), passing an argument containing the URL, name (if provided in the config object passed to $http), and the remaining requests. You can respond to these events in your application to suit your needs.

Additionally, there is a busy directive which can be declared as an attribute on an element to modify it in response to these events. The primary use case for this directive is the button element. Using this example, busy can swap the buttons's current HTML content with a message and disable it when busy.begin is received, and then restore the original content and re-enable to button on busy.end. It can also add or remove classes when either event is received. Other potential uses for this directive might be to show/hide or change the appearance of an element on either event.

Getting Started

Dependencies

The only required dependency is AngularJS (1.2 or later).

Files to Download

You can use the unminified angular-busy.js for development and minified angular-busy.min.js for production from here or install them from Bower by entering the command line bower install ng-busy.

Once you have the JS file, simply include it after AngularJS. 

<script src="angular.min.js"></script>
<script src="angular-busy.min.js"></script>

Configure

There is no additional configuration. The busyInterceptor will automatically push itself into the $http injectors.
